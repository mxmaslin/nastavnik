# Тестовое задание для fullstack

## Формулировка задания для кандидата

### Задание

Реализовать простой чат с пользователем, имитирующем проведение урока. 
Нужно подготовить объект - список вопросов с правильными ответами. Создай этот объект сам
Система задает пользователю вопрос(выбирается из списка вопросов по порядку) и принимает от пользователя ответ.
Эта пара вопрос - ответ отправляются в ml-сервис. в нем реализован эндпоинт fast-api с функцией-заглушкой, которая проверяет правильный ли ответ дал пользователь. 
Эта заглушка имитирует обращение к llm - запрос к llm занимает не менее 5 секунд. Эта функция возвращает 1 или 0, где 1 - это корректный ответ.
Если пользователь не давал ответа больше 30 секунд, то система записывает пустой ответ.
В 1 случае из 3 функция-заглушкаа возвращает код ошибки(llm-недоступна).
 
Вопрос-ответ - это пример учебного взаимодействия, записи о которых мы храним в таблице interaction_records в БД.
Нам очень важен порядок, корректность, какими были вопросы и ответы и дата и время взаимодействия.
Нам очень важно, чтобы все записи обязательно попадали в таблицу, даже такие, когда пользователь не ответил и закрыл урок или мы не смогли проверить корректность ответа.
Также нам важно, чтобы пользователь видел, что система рабоатет и что-то делает, пока он ждет следующий вопрос.

### Описание постановки

Реализован обучающий сервис, имитирующий проведение "урока", для которого заранее подготовлен упорядоченный список ответов. Многопользовательность не реализована т.к. не требовалась по ТЗ.

Пользователю через веб-интерфейс (Next.js) предлагается ознакомиться с текстом, после ознакомления - ответить на вопросы по этому тексту. Логика получения и обработки пользовательского ввода реализована на Django, DRF. Каждый ответ пользователя обрабатывается Celery-таской. Логика таски состоит в том, что на эндпоинт "ML-сервиса" направляется запрос, содержащий ID текущего вопроса пользователю, а также сам ответ пользователя. "ML-сервис" работает на FastApi и представляет собой асинхронную функцию-заглушку, которая, получив ID вопроса и ответ пользователя, "проверяет" правильность ответа. Правильность проверяется не за счёт обработки ответа ИИ-шкой (её в требуемом стеке нет), а путём проверки ответа пользователя в кэше Redis, и, в случае cache miss, асинхронного запроса в PostgreSQL через SQLAlchemy ORM. В этом запросе мы проверяем в БД, правильный ли ответ дал пользователь. Ответы пользователя, вне зависимости от их правильности / неправильности, сохраняются в БД в таблицу "interaction_record" вкупе с датой и временем ответа на вопрос урока.

Если пользователь не давал ответа больше 30 секунд (во время которых отображается уменьшающийся прогрес-бар этого тридцатисекундного периода ожидания), то сервис записывает пустой ответ. В одном случае из трёх функция-заглушка возвращает HTTP-код ошибки ("LLM недоступна"). Ожидание ответа пользователя не является блокирующим - он может попробовать завершить урок досрочно (в этом случае ему будет выведено сообщение, что все оставшиеся вопросы будут квалифицированы как неправильные), либо попробовать перескочить на следующий урок, не ответив на текущий (будет выведено сообщение, что для перехода к следующиему вопросу сначала надо ответить на текущий вопрос).

Либо перед, либо после прохождения урока пользователь может ознакомиться со статистикой прошлых уроков (процент успеха, время прохождения, был ли ответ пользователя успешно обработан "ML-сервисом" и т.п.).

Реализованы rate-лимиты от DDoS, код обучающего сервиса покрыт unit- и интеграционными тестами, сервисы развёрнуты через Docker Compose.

### Использованный стек

- Веб-интерфейс: Next.js
- Сервер статики: nginx
- Сервис приёма и обработки вопросов пользователя: Django, DRF, Celery, Redis, PostgreSQL
- "ML-сервис": FastAPI, Redis, PostgreSQL
- Тесты: Pytest
- Очередь: RabbitMQ
- Деплой: Docker Compose, Github CI
- Метрики: Prometheus, Grafana

### Запуск и просмотр

В корне проекта:

```bash
make up
```

| Сервис | URL |

| ML-сервис | http://localhost:8001 |
| Prometheus | http://localhost:9090 |
| Grafana (admin / admin) | http://localhost:3001 |
| RabbitMQ management | http://localhost:15672 (guest / guest) |
